### 1. 启动必读（按顺序，不可跳过）

Agent 每次对话开始或接收到新任务时，**必须先读以下文件才能行动**：

| 优先级 | 文件 | 读它干什么 |
|:--:|------|-----------|
| 1 | `AGENTS.md`（本文件） | 项目全局上下文、约定的束、Roadmap 当前进度 |
| 2 | `requires.txt` | 课程/任务原始要求——硬杠杠（时间、人数、不可念稿等） |
| 3 | `plan.md` | 研究计划——理论框架维度、方法、管线 |
| 4 | `content-guidelines.md` | 内容红线——必须有/可以有/不能有 |
| 5 | `brainstorm.txt` | 队长原始脑暴——约束、偏好、自相矛盾处 |
| 6 | 当前阶段对应的 SOP | 从 `docs/` 中按 Roadmap 当前状态选择 |

**读完后回答三个问题（心里回答，不输出给用户）：**
```
□ 我们现在在 Roadmap 的哪个阶段？下一个未完成的阶段是什么？
□ 当前焦点的阻塞是什么？用户卡在哪？
□ 我该用哪个 SOP？该 SOP 要求我先读哪些额外文件？
```

### 2. 引导式提问规则

Agent 与用户对话时，**必须遵守以下规则**：

| 规则 | 说明 |
|------|------|
| **不给开放式超纲问题** | 不问"你觉得深入研究什么方向好"——用户不知道才来找你 |
| **给选择题或带提示的填空题** | 每次提 3-5 个具体选项，推荐一项并说明理由 |
| **每次提问推动一个具体决策** | 不问空泛的，每次对话结束至少锁定一个决定（选题/维度/方法/分工） |
| **诊断优先于建议** | 先对照 checklist 找出缺口，再针对性提问——不问用户"你还缺什么" |
| **一次不超过 7 个问题** | 超过 7 个用户会懵，分轮次 |

**诊断矩阵**（每次不确定该问什么时对照此表）：

| 维度 | 问自己 | AGENTS.md 里有没有？ |
|------|--------|---------------------|
| 研究问题 | 是一句完整问句吗？是 Why 型吗？ | / |
| 选题范围 | 主题唯一吗？有没有多个候选没决定？ | / |
| 分析框架 | 有明确的维度名和数量吗？ | / |
| 方法具体性 | Survey 测什么？怎么测？多少样本？ | / |
| Why 链 | 从数据到结论的推理路径可想象吗？ | / |
| 团队分工 | 几个人？谁负责哪段？时间分配？ | / |
| 产出规格 | PPT 什么格式？还有什么提交物？ | / |
| 内容红线 | 什么必须说、什么不能做？ | / |

### 3. 行动守则

```
□ 所有依赖由 flake.nix 锁定 — 不在用户系统上 pip install 或 npm install -g
□ 所有产出物写入项目目录 — 不到 /tmp 写文件
□ 每步操作前查对应 SOP — SOP 自检清单不打勾不输出
□ Roadmap 状态实时更新 — 完成一个阶段立即改 ⬜ → ✅
□ 先生框架再统一生图 — 提纲/HTML/QA 全部落定后才一次性生成图表 PNG
□ PPT 是 outline 不是讲稿 — 每页最多 3-5 个关键词 + 1 张图，无完整句
□ 没读到 requires.txt 和 content-guidelines.md 之前 — 不开始生成任何汇报内容
```

## 协作工作流：三层递进（推荐）

<!--
  当队友交 .pptx 时，用此工作流加工。
  这不是强制的——如果队友直接交 ppt-outline.md 或 script.md，直接跳到 Layer 2。
-->

```
Layer 1: 提纲提炼（从队友输入到 md）
─────────────────────────────────
队友的 .pptx / .txt / 口语稿
  │ markitdown 或直接读取 → 提取内容
  ▼
ppt-outline.md    ← 每页梗概（关键词 + 图表位置）
                  ← 这一步审计：缺 Why？缺理论对照？补全

Layer 2: 内容补全（提纲 → 全稿）
─────────────────────────────────
ppt-outline.md
  ├→ script.md       ← 演讲文字稿
  ├→ qa.md           ← 预判 QA + 机动词汇
  └→ chart-data.json ← 图表数据（如有）

Layer 3: 统一生成（提纲 → HTML）
─────────────────────────────────
ppt-outline.md + 统一 style.css
  │ html-ppt skill
  ▼
index.html         ← 全组统一视觉
  │
  ▼
assets/*.png       ← 供 PDF/PPTX 嵌入
```

## 合并预览：`works/all/`

```
works/all/

├── merged.css       # 从各 Part 合并去重后的布局类

├── qa.md            # 跨 Part QA 汇集
└── assets/          # 所有 Part 的 PNG 汇集
```
