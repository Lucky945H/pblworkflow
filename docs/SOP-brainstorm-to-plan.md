# SOP：脑暴 → 可执行研究计划

**触发条件**：队长提交 `brainstorm.txt`（或口头/聊天记录转文字），说"我们脑暴完成了，帮我整理成计划"。

**目标**：把一堆散乱的 idea 变成一个像 `plan.md` 那样分节分明、可直接执行的文档。

---

## 一、输入：队长该交什么

最小输入（不能更少）：

| 必须包含 | 示例 |
|----------|------|
| 课程硬约束 | "来自《学术英语口语教程》5-8 单元，15 分钟，5 人，不能念稿" |
| 已选定的主题（至少方向） | "我们定了 College Life" |
| 研究方法偏好 | "要做 survey + interview + 教材理论" |
| 组内额外约束 | "不大量阅读、研究而非展示" |
| 当前想到的要点清单 | "1.期望和实际 2.原因 3.结论…" |

加分（有比没有好）：题型倾向（矩阵/单选/多选）、分析维度、样本量预期、是否允许访谈。

---

## 二、Agent 工作流

### 阶段 0：通读全部已有材料（必须先做）

如果工作区已有以下文件，**必须先全部读完**再开始：

| 文件 | 读它干什么 |
|------|-----------|
| `requires.txt` | 课程原始要求 — 时间、人数、不可念稿、500词摘要等硬杠杠 |
| `brainstorm.txt` | 队长提交的脑暴原始材料 |
| `AGENTS.md` | 项目约定、产出结构、协作工作流（如果有的话） |

### 阶段 1：解析 —— 从脑暴中提取结构化信息

读完输入后，用一个内部 checklist 逐项打勾：

```
□ 研究问题：是否已经是一句话问句？如果没有，候选表述是什么？
□ 分析维度：有没有提到"分几个 type/维度"？几个？叫什么？
□ 方法：Survey / Interview / Literal — 有没有说清各自干什么？
□ 数据来源：问卷测什么？访谈挖什么？理论从哪来？
□ 团队结构：几个人？怎么分工？每人几分钟？
□ 产出物：PPT、500词 Word、还有其他吗？
□ 约束：时间上限？不可念稿？不可有长文本 PPT？减少开放题？
□ 重心：队长有没有说"着重 why"还是"着重展示"？
```

**打勾后填入下表——空缺的就是接下来要问的。**

### 阶段 2：诊断 —— 识别模糊点和不一致

用以下诊断矩阵扫一遍。任何一个格子里信息缺失或矛盾，就是**下一轮提问的入口**。

| 维度 | 问自己 | 脑暴里有没有 |
|------|--------|------------|
| **研究问题** | 是一句问句吗？是 Why 型还是 What 型？ | / |
| **选题范围** | 主题是否唯一？有没有多个候选且没决定？ | / |
| **分析框架** | 用什么维度/lens 分析数据？叫得出名字吗？ | / |
| **方法具体性** | "做 survey"不够——测什么？怎么测？多少样本？ | / |
| **Why 链** | 从数据到结论的推理链是否可想象？还是"到时候看了数据再说"？ | / |
| **团队分工** | 几个人？谁负责哪段？时间分配出来了吗？ | / |
| **产出规格** | PPT 什么格式？500词什么内容？还有什么？ | / |
| **内容红线** | 什么必须说、什么不能做？有明确清单吗？ | / |

### 阶段 3：提问 —— 引导队长做决策

**提问原则**：
- 不给开放式超纲问题（"你觉得深入研究什么方向"→ 无效，他们就是不知道才来找你）
- 给选择题或带提示的填空题
- 每个问题都要推动一个具体决策
- 一次提 3-5 个问题，不要太少（反复来回效率低），也不要太多（超过 7 个队长会懵）

**按优先级分类的提问库**：

#### 第一优先级：选题锁定（如果脑暴里有多个方向）

```
"你列了 College Life、WeChat、Chinese vs Western Culture 三个方向。
 哪个是你们最终选定的？如果有两个都想要，哪个做主线？"
```

#### 第二优先级：研究问题定型

```
当前候选研究问题是"XXX"。但这不是一个完整的 Why 型问句。
建议改为 "Why couldn't students reach their expectations of college life on campus?"
这个表述：
- 是 Why 型（不是 What/How）
- 限定了群体（students on campus）
- 可操作化（expectation vs reality → measurable gap）
用这个，还是你们有别的想法？
```

#### 第三优先级：维度/框架补全（最容易空白的格）

```
你提到"分几个 type"和"几个维度各自的期望"——
但没说具体哪几个维度。给我候选列表，我来帮你们判断是否正交、是否全面。
目前从脑暴中能提取的：学术、社交、时间管理/作息、消费/经济。
还缺什么？
- 个人兴趣发展？
- 身体健康？
- 职业发展（实习/就业）？
- 心理健康？
- 校园生活环境（宿舍/食堂）？
选你们想覆盖的。
```

#### 第四优先级：方法操作化

```
"做 survey"需要落地：
- 问卷结构：基本信息 → 期望测量 → 现实测量 → 原因多选 → 改善建议。同意吗？
- 每个维度用矩阵题 1-5 量表，期望和现实用相同量表方便做差值。OK 吗？
- 预期回收多少份？
- 访谈几个人？什么问题？怎么选受访者？
```

#### 第五优先级：团队与管线

```
- 5 个人各自负责汇报的哪个环节？每人几分钟？
- 整个 15 分钟的汇报顺序是什么？
- 需要 500 词英文摘要，谁来写？什么格式？
```

#### 第六优先级：内容红线

```
汇报硬杠杠我帮你从课程要求中提炼：
- PPT 只能有关键词+图表，不能有完整句子←这个最容易翻车
- 不能逐题念问卷结果 ← 另一个常见死法
- 重心是 Why，不是描述现状
- 结论必须回应开头的研究问题
以上我对，还是你们有补充？
```

### 阶段 4：整合 —— 生成 plan.md

当所有格都填好了，生成 `plan.md`。**必须包含以下章节，缺一个都不行**：

```
1. 我们要做什么         ← 一句话研究问题 + 产出物清单
2. 重心                 ← 用一个比喻讲清楚 "唯一重心是 Why"
3. 具体内容和如何获得   ← 一张表：5 个环节 × (内容/如何获得/核心或扩展)
4. 方法详解             ← Survey 设计逻辑 / Interview 设计 / 理论来源
5. 核心 vs 扩展         ← 两张清单：必须有 vs 锦上添花
6. 整体基调             ← 三组 应该是 vs 不该是 的对比表
                        （研究v展示、PPT规范、Findings写法、数据用法）
7. 汇报逻辑线速览       ← 时间线 + 每人一句话台词模板
```

## 三、真实案例：brainstorm.txt → plan.md 的转换

### 输入摘要

`brainstorm.txt` 里有什么：
- 多个选题方向（WeChat、东西文化、College Life×4）
- 最终选定 "why couldn't students reach their expectations"
- 约束：研究而非展示、不大量阅读、survey+interview+literal
- 模糊要点："分几个 type"、"option 选几个就好"、"知道该怎么改善但是 why 不改善？"
- 自相矛盾："减少开放题" 但问卷结构偏开放

### Agent 识别出的缺口

| 格 | 状态 | 动作 |
|----|------|------|
| 研究问题 | 半成型 | 定型为一句 Why 型问句 |
| 选题 | 已定 | OK |
| 维度 | **空白** | 从脑暴中提取候选 + 补全为 5 维度框架 |
| 问卷结构 | 模糊 | 设计 4-block 问卷逻辑 |
| 访谈 | 仅提到"可以做" | 给出 3 个核心 Why 问题 |
| 团队分工 | **空白** | 按 5 人 × 3min 分 5 个环节 |
| 时间管线 | **空白** | 输出 0-15min 精确时间线 |
| PPT 规范 | **空白** | 从课程要求提炼内容红线 |
| 内容红线 | 仅"着重 Why" | 扩展为 应该是vs不该是 对照表 |

### 产出

`plan.md`：7 节，每一节都从脑暴中的一句话展开为可执行方案。队长拿到后可以直接分发给组员，各人知道自己负责什么。

## 四、SOP 自检清单（Agent 在输出 plan.md 前自问）

```
□ 这个计划能不能直接发到小组群里，5 个人看完都知道自己该干嘛？
□ 研究问题是不是一句完整英文问句（Why 型）？
□ 分析方法（维度/框架）是不是有具体名字和数量？
□ Survey 的 block 结构是不是写到了题目类型级别（矩阵/单选/多选）？
□ Interview 有没有至少 2-3 个核心问题的示例？
□ 时间线有没有精确到分钟的 5 段划分？
□ 内容红线有没有三个以上的反例（不该做 / 应该做对照）？
□ 自相矛盾的指令有没有被识别和澄清？（如果有则必须先问再写）
```

全部打勾 → 输出 plan.md。

---

## 五、输出后 — 交接给下一个 Agent

`plan.md` 生成完毕后的下一步通常是：

1. **问卷设计**：根据 plan 中的 4-block 结构写出 `survey-wjx.txt`（问卷星导入格式）
2. **访谈提纲**：根据 plan 中的核心问题扩展出完整半结构化访谈指南
3. **如果 plan 和课规有冲突**：优先遵守课规，回来改 plan

这些步骤在 `AGENTS.md` 的 Roadmap 中有定义。SOP 到此结束，agent 应写一句总结，明确下一阶段的 entry point。
