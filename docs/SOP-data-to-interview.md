# SOP：调查数据 → 分析报告 → 访谈设计

**触发条件**：问卷已回收，拿到了 问卷星导出的 CSV（如 `统计结果.csv`），需要从原始数据提炼出可汇报的发现，并将发现反馈到访谈提纲设计中。

**目标**：三步走——（1）解析 CSV 产出量化分析报告；（2）将分析结果结构化为一套「Why 解释链」；（3）用解释链中的缺口反向设计访谈问题，精准追问数字背后的机制。

---

## 一、输入：Agent 必须先读什么

| 文件 | 读它干什么 |
|------|-----------|
| `统计结果.csv`（或 .xlsx） | 原始数据 — 问卷星导出的宽表，每行一个受访者 |
| `plan.md` | 理论框架维度清单、Block 结构、研究问题 — 分析维度必须对齐框架 |
| `survey-wjx.txt` | 问卷原文 — 核对题号、选项文本、矩阵维度映射 |
| `content-guidelines.md` | PPT 红线 — 反向约束分析输出（哪些数字适合上 PPT，哪些不适合） |
| `analysis/`（如果有旧版） | 已有分析产物 — 增量分析不要覆盖，追加或重建 |

## 二、Agent 工作流

### 阶段 0：解析 CSV 列索引（必须做）

问卷星导出的 CSV 有 90 列是正常的。每个多选题的每个选项都是一列（binary 0/1），题目题干本身可能也是一列（skip 标记）。

**必须做的步骤：**
```
□ 数 header 有多少列
□ 逐列打印 header → 每一列对应问卷的哪个题、哪个选项
□ 选 2-3 行数据打印关键列的值 → 验证"你以为的 col 74 真的是 Q12"
□ 特别注意：问卷星矩阵题的题干列可能是虚拟的 skip 标记（如 Q9 col 49），
   真正的选项从下一列开始
```

**本次 session 翻车教训：**
- 最初以为表格是 89 列，实际是 90 列。问卷星在 Q9 前加了一个 skip 条件列（col 49）。
- 这导致 Q9-Q15 全部列索引错位 1 格，一度输出「Freshman: 0%」「Q12 只有一项有数据」。
- **解决方案**：先不写分析脚本，先用一个诊断脚本打印前 3 行的列号→值映射，人工确认。

### 阶段 1：基础描述统计

必须出的数字（每项都要有 count + percentage）：

| 题号 | 内容 | 输出 |
|------|------|------|
| Q1-Q2 | 人口学 | 年级分布、专业分布（饼图或频数表） |
| Q3 vs Q5 | 期望 vs 现实矩阵 | 每个维度的 **均值差值**（期望均值 - 现实均值）→ 这是汇报核心 |
| Q6 | 总体符合度 | 5 级分布（Fully matches → Fully doesn't match） |
| Q7 | 感知最大差距维度 | 多选题频数 + 百分比 |
| Q8 | 达到期望的维度 | 多选题频数 + 百分比 |
| Q11 | 差距原因 | 多选题频数 + 百分比（11 个原因排序） |
| Q10 | 情绪反应 | 多选题频数 + 百分比 |
| Q14 | 想要什么帮助 | 多选题频数 + 百分比 |
| Q12 | 是否尝试缩小差距 | 4 级分布 |
| Q13 | 为什么没效果/没行动 | 多选题频数 + 百分比 |
| Q4 | 期望来源 | 多选题频数 + 百分比 |
| Q9 | 达到期望靠什么 | 多选题频数 + 百分比 |
| Q15 | 开放题 | 清洗后列出所有非空回答 |

**代码放在 `analysis/analyze.py`**，输出写入 `analysis/raw_stats.txt`。

### 阶段 2：交叉分析（分层对比）

按至少两个维度做交叉：

#### 2a. 分年级

```
对每个维度：各年级的 gap（期望-现实均值）
输出：一张表，行=维度，列=年级
```

**注意：** 非 Freshmen 的子样本通常很小（如 sophomore n=8, junior n=4, senior n=5）。即使趋势明显，汇报时也必须标注「small sample, suggestive only」。

#### 2b. 分专业

```
对每个维度：各专业的 gap
输出：同上
```

**本次发现示例：** STEM 学生的 Career Development gap (+1.39) 显著高于平均水平 (+1.12)，这是设计访谈 Part C 的依据。

#### 2c. 可选：分 Q12 响应

比较「Tried & improved」vs「Tried & failed」两组的 gap 差异，看成功者特征。

### 阶段 3：合成 Why 解释链（最关键的步骤）

量化数据不能只是"数字罗列"，要组合成一条因果链。方法：

#### 步骤 1：找出最大 gap 维度

按 gap 排序，取前 3。**但不要只看排序**——还要交叉看 Q7（学生主观感知）和 Q8（满意度最低的）。

```
本次发现：
  实测最大 gap: Time Autonomy (+1.27)
  感知最大 gap: Academic Learning (80.2% in Q7)
  满意度最低:    Career Development (7.4% in Q8)

→ 三重聚焦：时间管理是隐藏真凶，学术是表面症状，职业是长期焦虑
```

#### 步骤 2：配对「gap 维度 ↔ Q11 原因」

每个高 gap 维度，查 Q11 中对应的原因选项的频率。建立映射：

| 高 gap 维度 | Q11 对应原因 | 频率 |
|-------------|-------------|------|
| Time Autonomy +1.27 | 缺乏时间管理和自律能力 | 58.0% |
| Career +1.12 | 缺乏职业规划和实习机会 | 32.1% |

如果某个高 gap 维度在 Q11 中没有高频原因对应 —— 说明受访者不知道为什么有 gap —— 这是访谈的**高优先级追问点**。

#### 步骤 3：建立三层因果链

```
Layer 1: 近因 — 学生直接归因（Q11 高频选项）
Layer 2: 期望来源 — 为什么不现实（Q4 高频选项）
Layer 3: 执行障碍 — 为什么知道却做不到（Q12+Q13 交叉）
```

**输出格式**：`analysis/analysis_report.md`，其中 `Synthesized Answer to "Why?"` 节必须是链条式的（不能是 bullet list 罗列）。

#### 步骤 4：识别反直觉发现

反直觉 = 两个数据来源对同一问题的结论矛盾。这是 PPT 中最有价值的分析点。

```
本次反直觉发现一览表：

| 反直觉 | 来源 A | 来源 B | 解读 |
|--------|--------|--------|------|
| 学生认为学术差距最大 | Q7 80.2% Academic | 实测 Time +1.27 > Academic +1.09 | 时间管理崩溃表现为学术表现差，学生归咎于课程难度 |
| 大一 gap 不是最大的 | 常识：大一最不适应 | 实测：Sophomore gap +1.33 > Freshman +1.02 | 二年级低谷——从"适应焦虑"变成"意义焦虑" |
| 大四情感 gap 最严重 | 常识：大四最成熟 | 实测 Emotional +1.60 是全校最高 | 功能适应了但人更孤独——实用性适应≠情感适应 |
```

### 阶段 4：从分析发现 → 设计访谈模块

**原则：** 访谈不是重新测一遍问卷，而是追着分析中解释不了的缺口、反直觉点、和高频原因背后的"具体机制"走。

#### 4a. 定义「访谈 Profile」

从数据分析中提取需要深度追问的人群类型。每个 profile 必须有一个**明确的量化依据**（不是凭感觉）。

| Profile | 量化依据 | 追问重点 |
|---------|---------|---------|
| Freshmen | n=64 (79%)，Time gap 最大 | 从结构化高中到无结构大学的具体过渡体验 |
| STEM students | Career gap +1.39 vs avg +1.12 | 理工科为什么职业焦虑更严重？专业壁垒 vs 就业市场 |
| Sophomores | Gap +1.33 全校最高 | 二年级的失望为什么比一年级更重？从"不会管时间"变成"不知道意义在哪" |
| Seniors | Emotional gap +1.60 全校最高 | 什么都有了但更孤独？情感需求的结构性缺失 |
| Tried & failed (61.7%) | 知道方法但做不到 | 执行失败的精确机制——卡在哪一步 |
| Tried & improved (16%) | 少数成功者 | 他们做对了什么？可复制的策略 |
| High fantasy-expectation | Q4 自我想象 81.5% | 那幅"理想大学画面"长什么样？从哪来的？ |

#### 4b. 为每个 Profile 写 3-5 个追问

追问规则：
- 第一问：建立 narrative（"给我讲一个故事"） — 开放式
- 中间问：切入具体机制（"那你试了什么？为什么卡住了？"） — 半结构
- 最后问：离题开放式（"如果有一件事你能改变…"） — 保证金句

#### 4c. 设计通用模块

两个模块适用于所有人：
- **Opening (Part A)**：建立个人 gap 叙事，3-4 个热身问题
- **Closing (Part H)**：期望工厂 — 追问期望来源和心理图景

#### 4d. 生成访谈目标矩阵

| Profile | 样本目标 | 主模块 | 选配模块 |
|---------|:---:|--------|---------|
| Freshman, STEM | 2-3 | A+B+F | H |
| Freshman, non-STEM | 1-2 | A+B+H | |
| Sophomore | 2 | A+D+F+H | |
| Junior | 1-2 | A+C+D+H | |
| Senior | 1-2 | A+C+E+H | |
| "Improved" | 1-2 | A+G+H | |
| "Failed" | 1-2 | A+F+H | |

**总目标**：8-12 人。每人选 2-3 个模块，10-15 分钟。

### 阶段 5：输出文件

```
analysis/
├── analyze.py          ← 主分析脚本（可复跑）
├── raw_stats.txt       ← 基础统计全文
├── cross_analysis.py   ← 交叉分析脚本
├── cross_tabs.txt      ← 分年级/专业的 gap 表
├── chart_data.json     ← 图表 JSON（可直接喂给 chart-visualization）
└── analysis_report.md  ← 综合分析报告（含因果链 + PPT 建议）

interview-guide.md      ← 完整访谈提纲（8 模块 × 40+ 题 + 矩阵）
```

- `plan.md`（5 维框架）
- `survey-wjx.txt`（15 题）

### Agent 做了什么

#### Step 1 — 列索引诊断

第一版脚本 89 列 → 输出「所有年级 0%」「Q12 仅 24 人有数据」。诊断发现实际 90 列，Q9 前有 skip 标记列，导致 Q9-Q15 全部偏移。

**修复**：写诊断脚本打印前 3 行的列→值映射，重新建立正确索引。

#### Step 2 — 基础统计

13 项分析输出到 `raw_stats.txt`，所有频数和百分比核对无误。

#### Step 3 — 交叉分析

分年级、分专业计算各维度 gap。

**关键发现**：
- Sophomore gap 全校最高 (+1.33)，而不是 Freshman (+1.02)
- Senior 的 Emotional gap 全校最高 (+1.60)
- STEM Career gap (+1.39) 显著高于平均

这三个发现直接催生了访谈 Part C、D、E。

#### Step 4 — 合成 Why 链

三个关键配对：
1. Time Autonomy gap +1.27 ↔ Q11「缺乏时间管理」(58.0%) → 时间陷阱
2. 61.7% tried & failed ↔ Q13「知道但缺乏执行力」(48.1%) + 「缺外部支持」(46.9%) → 执行双重困境
3. 81.5% self-imagination + 72.8% social media ↔ Q11「不现实期望」(53.1%) → 期望工厂

这三个配对构成了一条完整因果链：**幻想级期望 → 无结构自主冲击 → 独力尝试失败 → 习得性接受**。

#### Step 5 — 反直觉发现提炼

4 组反直觉对比（见上文阶段 3 步骤 4），每组都对应到访谈的特定模块。

#### Step 6 — 访谈设计

不重做，而是从分析结果**反向推导**需要追问什么：

```
分析发现                                   → 访谈模块
─────────────────────────────────────────────────────
Time Autonomy 是#1 实测 gap (但学生不觉得) → Part B：追问"为什么你不觉得？
                                            你以为的'自由'和实际的'自由'有什么区别？"
61.7% tried & failed                      → Part F：追问"你试了什么？卡在
                                            哪一步？试了几次之后放弃了？"
Sophomore gap 反而更大                    → Part D：追问"第一年和第二年不
                                            满意的部分变了没有？从什么变成了
                                            什么？"
Senior 情感 gap 全校最高                  → Part E：追问"你什么都适应了——
                                            时间、钱、课业——但为什么更孤独？"
Career 仅 7.4% 满意                       → Part C：不限年级追问"你对职业的
                                            期待从哪来？为什么觉得学校帮不上？"
Self-imagination 81.5%                   → Part H：通用闭合题追问"那幅画
                                            面长什么样？谁画的？"
```

- `analysis/` 6 个文件
- `interview-guide.md`（8 模块，40+ 题，按 profile 组合）
- `plan.md` 的访谈部分同步更新

## 四、SOP 自检清单（Agent 在输出 analysis_report.md 和 interview-guide.md 前自问）

```
□ CSV 列索引是否用诊断脚本验证过？（不能凭 header 猜测）
□ 每个分析输出的 count + percentage 是否正确？（样本量 n 是否一致？）
□ Q3 和 Q5 的 gap 计算是否一一对应？（矩阵平行性）
□ 是否找到了至少 2 组反直觉发现？（两个数据来源结论矛盾）
□ 是否输出了因果链而不仅仅是频数表？
□ 每个访谈 Profile 是否都有量化依据？（不能凭直觉说"问一下大一学生"）
□ 访谈模块的数量是否对应分析中的缺口数量？（缺口=分析无法解释的点）
□ Q15 开放题是否清洗过？（空/"无"/"跳过"/单字/纯数字要过滤）
□ `analysis_report.md` 的 Why 节是否是链条式而非列表式？
□ `chart_data.json` 的字段名是否与 chart-visualization 技能兼容？
```

全部打勾 → 输出 `analysis/` 全套 + `interview-guide.md`。

## 五、常见翻车点与应对

### 翻车 1：问卷星 CSV 列数和你算的不一样

**症状**：某个维度的均值明显不合理（如 Academic Adaptation 期望 1.7）或者全是 0。
**诊断**：别猜，打印 header + 前 3 行的 key-value。
**根因**：问卷星的多选题每个选项是独立列，跳题逻辑会产生额外标记列。
**修复**：重新建立列索引映射。

### 翻车 2：访谈设计像问卷重问一遍

**症状**：访谈题和问卷题几乎一样（"你觉得学术差距大吗？"→ Q7 已经答过）。
**正确做法**：访谈只追问问卷**无法回答**的问题——机制、叙事、情感细节。
- ❌ "What was your expectation before college?" → 问卷 Q3 已测
- ✓ "Tell me about the moment you realized college wasn't what you expected. What were you doing?"

### 翻车 3：分析报告变成数字列表

**症状**：「70% 的人选了 A，58% 的人选了 B，43% 的人选了 C…」
**正确做法**：每个数字都要接一句"这意味着…"、"这和另一个数字的矛盾在于…"。

### 翻车 4：忽略小样本警告

**症状**：Sophomore n=8 的 gap 和 Freshman n=64 的 gap 并排展示，不加标注。
**正确做法**：小样本（n<15）的趋势必须标注"suggestive, small sample"。可以作为访谈追问方向但不能作为结论。
