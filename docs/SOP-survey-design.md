# SOP：研究计划 → 可发放问卷

**触发条件**：`plan.md` 已定稿（研究问题 + 理论框架 + 方法明确），需要设计具体问卷。

**目标**：把计划中的「Survey 测什么维度」落地为一份正交、全面、可导入问卷星的问卷。

---

## 一、输入：Agent 必须先读什么

| 文件 | 读它干什么 |
|------|-----------|
| `plan.md` | 理论框架维度、Block 结构意图、预期样本量、内容红线 |
| `brainstorm.txt` | 队长原始约束（如"减少开放题""大 type 不要太多"） |
| `content-guidelines.md` | PPT/汇报红线 — 反向约束问卷维度数（维度太多汇报讲不完） |
| `survey-wjx.txt` | 如果是修订已有问卷，必须逐题比对 |

---

## 二、Agent 工作流

### 阶段 0：提取维度清单（从 plan.md）

从 plan.md 的理论框架节提取**维度名称和数量**。记下来：

```
维度1: Academic Adaptation
维度2: Career Development
维度3: Social & Emotional
...
共 N 个维度
```

这是本次问卷设计的**唯一上游**。问卷的每一个测量项都必须能归入其中一个维度。

---

### 阶段 1：设计 Block 结构

问卷必须分 Block，每个 Block 职责单一。标准 4-Block 结构：

```
Block 1 — 基本背景
  年级 / 专业（2 题，单选）
  作用：后续分层对比用

Block 2 — 期望测量
  对每个维度的入学前期望程度（矩阵题，1-5 量表）
  作用：建立 baseline

Block 3 — 现实测量
  对每个维度的实际体验程度（矩阵题，1-5 量表，与 Block 2 平行）
  作用：与 Block 2 做差值 → 量化差距

Block 4 — 归因 + 改善
  原因多选题（直接回答 Why）
  情感反应题（心理状态）
  行动与障碍题（知道但做不到？）
  改善需求题（想要什么帮助？）
  最多 1 个开放题（内容红线强制）
```

附加题（非必需但推荐）：
- 期望来源题（Block 1 之后）：影视剧/高中老师/社交媒体 — 解释「不现实期望」的来源
- 总体符合度题（Block 3 之后）：单题概述，验证矩阵均值的可靠性

---

### 阶段 2：维度 → 矩阵题项映射（关键步骤）

#### 规则 1：每个维度至少 1 个矩阵题项

| 维度 | 题项示例 |
|------|---------|
| Academic Adaptation | 学到深度专业知识与技能 |
| Career Development | 获得实习机会，明确未来职业方向 |
| Social & Emotional | 结交志同道合的朋友 / 建立恋爱或亲密关系 |
| Autonomy & Self-management | 自由安排作息与时间 / 经济消费自主管理 |
| Personal Growth & Well-being | 发展个人兴趣爱好 / 规律锻炼保持健康 |

#### 规则 2：单维度内题项数尽量均衡

如果 Financial 仅 1 项而 Self-management 有 3 项，合并为均分或加权时会放大不平衡。理想状态每个维度 1-2 项。

#### 规则 3：Block 2 和 Block 3 的题项必须一一对应

题项 text 可以略有区别（Block 2 "你对…的期望" vs Block 3 "你对…的实际体验"），但**顺序、数量、覆盖的维度必须完全相同**。这样才能做差值。

---

### 阶段 3：设计归因题（直接回答 Why）

这是问卷的核心——不是测「差距多大」，而是测「为什么有差距」。

#### Q11 设计原则（原因多选题）

- 每个框架维度至少对应 1 个原因选项
- 加上跨维度通用原因（如信息差、过渡适应）
- 各选项之间不能有概念交集

**映射表（必须做）：**

| 框架维度 | Q11 对应选项 | 是否正交？ |
|----------|------------|-----------|
| Academic Adaptation | B. 课程难度或教学方式与预期差异大 | ✓ |
| Career Development | J. 缺乏职业规划和实习机会 | ✓ |
| Social & Emotional | D. 社交环境变化大… / K. 恋爱关系未达预期… | ✓ |
| Autonomy & Self-management | C. 缺乏时间管理和自律能力 + E. 经济压力超出预期 | ✓ |
| Personal Growth & Well-being | H. 活动平台有限… / I. 缺乏运动氛围… | ✓ |
| 通用 | A. 对大学了解不足 / F. 过渡缺乏引导 / G. 心态变化 | ✓ |

#### 交集审计（必须做）

两两阅读每个选项，问：**这个选项的含义是否能被另一个选项覆盖？** 如果能，就是交集，需要改措辞。

真实案例（本次 session）：

| 原选项 | 原选项 | 交集 | 修复 |
|--------|--------|------|------|
| H. 校园资源有限，难以发展个人兴趣爱好 | K. 校园硬件设施及住宿条件低于预期 | 「校园资源」涵盖「硬件设施」→ 口径模糊 | H 改为「活动平台或社团有限」，K 与用户确认后删除 |
| H. 校园资源有限，难以发展个人兴趣爱好 | I. 缺乏运动环境，忽略了身体健康 | 运动可以是一种兴趣爱好 → 「运动环境」可能与 H 重叠 | I 改为「缺乏运动氛围或同伴激励」，聚焦 social 层面 |

---

### 阶段 4：互斥性检查

多选+单选混排时，检查所有逻辑冲突：

| 问题 | 检查点 | 修复 |
|------|--------|------|
| Q8 有「以上均未达到期望」 | 与其他选项互斥，但多选题不天然支持互斥 | 问卷星后台设互斥；或改为单选题 |
| Q12→Q13 跳转 | 只有选了 B/C 才应出现 Q13 | 补充显式跳过说明或在问卷星设跳题逻辑 |
| Q9 条件题 | 如上一题选了「均未达到」，此题应跳过 | 写清跳过条件 |

---

### 阶段 5：维度与理论框架对齐审计（最终把关）

**在问卷定稿前，必须做最后一次整体审计。** 这是本次 session 的核心经验。

#### 审计清单

```
□ 问卷矩阵覆盖的维度数：______
□ 理论框架的维度数：______
□ 两者是否一致？如果不一致，哪个维度多了/少了？
□ 多了的：问卷有但理论框架没有 → 汇报时无法用框架解释 → 要么删问卷项，要么扩框架
□ 少了的：理论框架有但问卷没测 → 汇报时有框架无数据 → 要么补问卷项，要么缩框架
□ 所有维度名是否在问卷 Q7/Q8/Q11/Q14 中有一致体现？
□ 删掉某个维度后，Q7/Q8/Q11/Q14 是否需要重新编号/去重？
```

#### 本次 session 真实审计结果

| 问题 | 发现 | 处理 |
|------|------|------|
| 框架维度 vs 问卷维度不匹配 | 框架 4 维，问卷 9 项矩阵（含校园环境），映射不上 | 用户决定：删校园环境，扩展框架至 5 维 |
| 某维度题项过多/过少 | Social & Emotional 含 2 项，Financial 仅 1 项 | 用户接受 1-2 项均衡，不做调整 |
| 跨题一致性 | 删校园环境后，Q7/Q8/Q14 均有对应选项需同步删除 | 逐题删除 + 重新编号 |

---

### 阶段 6：生成问卷星导入文本

格式约定：
- 每题标明 `[单选题]` / `[多选题]` / `[矩阵题]` / `[多项填空题]`
- 矩阵题的 scale 写在题号下一行（如 `很低 较低 一般 较高 很高`）
- 矩阵题项逐行列出，不加编号
- 选项用 `A.` `B.` 等字母编号
- 跳题逻辑用括号注明（如 `（如上一题选了"以上均未达到"，可跳过）`）

最终输出 `survey-wjx.txt`，可直接复制粘贴到问卷星导入。

## 三、队长反馈的常见模式与应对

### 模式 A：「扩充维度」

队长说：「框架加几个维度」 → 意味着问卷矩阵也要加题项。
- 先确认新维度名称和定义
- 再为每个新维度添加对应的矩阵题项（Block 2+3 平行）
- 同步更新 Q7/Q8/Q11/Q14 的所有相关选项
- 更新 plan.md 和 content-guidelines.md 中的维度引用

### 模式 B：「这个维度不要了」

队长说：「删掉 XX 维度，我们 focus 在 YY」→ 做的是减法，但容易漏。

**删除维度的完整检查清单：**
```
□ Block 2 矩阵：删除题项
□ Block 3 矩阵：删除对应题项
□ Q7 差距认知：删除对应选项 + 重新编号后续选项
□ Q8 达到期望：删除对应选项 + 重新编号后续选项
□ Q11 归因：删除对应原因选项 + 重新编号
□ Q14 改善需求：删除对应选项 + 重新编号
□ plan.md：删除维度引用
□ content-guidelines.md：删除维度引用
```

漏掉任何一处，问卷就会出幽灵选项（在其他题出现但矩阵没有的维度）。

### 模式 C：「选项 A 和 B 好像有重叠」

队长说：「这两个是不是差不多？」→ 交集审计。
- 精确读取两个选项的措辞
- 判断：A 的外延是否包含 B？B 的外延是否包含 A？两者是否有共同子集？
- 如果有重叠，修改措辞使两者正交（如本次 H↔I、H↔K 修复）
- 修改后再次确认：现在两个选项是否不再有交集？

### 模式 D：「大 type 不好，但我不确定怎么改」

队长对某个设计方向不满意但说不出具体方案 → 给出候选方案列表。
- 不要求队长自己想出完美方案（那是 Agent 的工作）
- 给 2-3 个具体方案，附带各方案对汇报的影响
- 选项以选择题形式给出

## 四、SOP 自检清单（Agent 在输出 survey-wjx.txt 前自问）

```
□ Block 2 和 Block 3 的矩阵题项一一对应、顺序一致吗？
□ 每一个矩阵题项能对应到 plan.md 中的一个理论维度吗？
□ 每个理论维度在 Q11 中至少有一个归因选项吗？
□ Q11 各选项之间有没有概念交集？（两两读一遍）
□ 互斥选项（如「以上均未达到」）有没有标记跳转逻辑？
□ Q7/Q8/Q11/Q14 的选项字母编号在删除/添加后是否连续无跳跃？
□ 开放题不超过 1 个吗？
□ 问卷维度数与理论框架维度数一致吗？
□ 如果删了维度，所有相关题号都同步修改了吗？
□ 队长在 brainstorm 中的约束（如"减少开放题""大 type 不要多"）都满足了吗？
```

全部打勾 → 输出 `survey-wjx.txt`。

## 五、真实案例：本次 session 的 dimension 对齐审计

### 背景

已有 `survey-wjx.txt`（15 题，9 维矩阵，已发放）。队长要求检查组内理论框架（当时 4 维）与问卷的一致性。

### Agent 审计过程

**Step 1 — 映射矩阵**：把 9 个问卷题项逐一尝试归入 4 维框架。

| 问卷题项 | 能归入哪个框架维度？ | 问题 |
|---------|-------------------|------|
| 学到专业知识 | Academic ✓ | |
| 结交朋友/社交圈 | Social ✓ | |
| 自由安排作息 | Self-management ✓ | |
| 经济消费自主 | Financial ✓ | |
| 发展兴趣爱好 | Self-management? | 兴趣≠时间管理，勉强 |
| 规律锻炼 | Self-management? | 同上 |
| 职业方向 | Academic? | 职业规划≠学术学习 |
| 校园生活环境 | ❌ | 无法归入任何维度 |
| 恋爱/亲密关系 | Social? | 可归入但性质不同 |

**Step 2 — 交集审计（Q11）**：发现 H/I/K 三选项概念重叠。

**Step 3 — 向队长提问**：

> 「校园生活环境无法归入现有框架，建议删除。余下 8 项可归入 5 维框架。同意吗？」

队长回应：删校园环境，扩框架，修复交集。

### 修改链条（完整）

```
用户指令：「扩充plannn的维度 + 删校园环境 + Q11 H/I/K去交集」
       ↓
plan.md         ← 框架 4→5 维；Block 2/3 从 4 项→8 项
content-guidelines.md  ← 维度示例更新；图表说明改"各维度并排"
survey-wjx.txt  ← 5 处修改（矩阵-1项 / Q7重编号 / Q8重编号 / Q11去重叠+删K / Q14删I）
       ↓
验证：Block2(8项) = Block3(8项) ✓ / 5维框架覆盖8项 ✓ / H/I/K无重叠 ✓
```

## 六、输出后 — 交接给下一个 Agent

问卷定稿后的下一步：
1. **导入问卷星** — 复制 `survey-wjx.txt` 到问卷星批量导入
2. **设计访谈提纲** — 从 Q11 的原因选项中挑选适合深度追问的维度，扩展为半结构化访谈问题
3. **数据分析计划** — 预设期望 vs 现实的差值计算方式、维度内多题项的聚合方式（均分/加权/中位数）
